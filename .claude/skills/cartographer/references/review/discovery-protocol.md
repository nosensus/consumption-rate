# Atlas Refinement - Discovery Protocol

This reference defines the atlas refinement workflow for incremental pattern learning during reviews.

## Purpose

During `/navigator:review`, track discoveries that should improve the atlas:
- New patterns not yet documented
- New anti-patterns discovered during review
- Missing conventions the code follows but atlas doesn't capture
- Unclassified files that don't match any pattern

## Workflow

### 1. Detect Undocumented Patterns

Scan implementation for:

| Discovery Type | What to Look For |
|----------------|------------------|
| New patterns | File structures/conventions not in schema.yaml |
| New anti-patterns | Mistakes that should be documented for others |
| Missing conventions | Rules the code follows but atlas doesn't capture |
| Unclassified files | Files that don't match any pattern in atlas |

### 2. Record Discoveries

For each discovery, record:

| Field | Description |
|-------|-------------|
| `type` | `pattern` \| `anti_pattern` \| `convention` |
| `description` | What was observed |
| `evidence` | File paths and line numbers |
| `suggested_pattern_id` | Which pattern section it belongs to |

### 3. Write to Staging File

**Location:** `.claude/skills/atlas/staging/discoveries.yaml`

```yaml
# Atlas Discoveries - Pending Review
# Generated by /navigator:review on {timestamp}
# Run /cartographer:capture --from-staging to process

discoveries:
  - type: anti_pattern
    pattern_id: controllers
    description: "Direct database queries in controller"
    evidence:
      - file: src/controllers/UserController.ts
        line: 45
        snippet: "db.query('SELECT * FROM users')"
    suggested_rule: "Don't query database directly in controllers - use provider layer"

  - type: convention
    pattern_id: providers
    description: "All providers use dependency injection constructor pattern"
    evidence:
      - file: src/providers/UserProvider.ts
        line: 10
      - file: src/providers/AuthProvider.ts
        line: 8
    suggested_rule: "Inject dependencies via constructor, not inline imports"

  - type: pattern
    pattern_id: null  # New pattern
    description: "Background job handlers follow consistent structure"
    evidence:
      - file: src/jobs/EmailJob.ts
      - file: src/jobs/ReportJob.ts
    suggested_addition:
      id: background_jobs
      keywords: [job, worker, queue, background, async]
      file_convention: "src/jobs/{Name}Job.ts"
```

**Create staging directory if needed:**
```bash
mkdir -p .claude/skills/atlas/staging
```

### 4. Prompt for Capture

**If discoveries found, use AskUserQuestion:**

```
📚 Atlas Learning Opportunity

I discovered {count} items that could improve the atlas:
- {count} new anti-patterns
- {count} new conventions
- {count} potential new patterns

Options:
```

| Option | Action |
|--------|--------|
| "Capture all now" | Run `/cartographer:capture --from-staging` |
| "Review staging file first" | Show path to staging file |
| "Skip for now" | Continue without capturing |
| "Never suggest during reviews" | Add preference to atlas |

### 5. Capture Confirmation

**If "Capture all now":**
```
Capturing discoveries to atlas...

Added to schema.yaml:
- patterns.controllers.anti_patterns_summary: +1 item
- patterns.providers.conventions: +1 item
- patterns.background_jobs: NEW pattern

Staging file cleared.
```

## Discovery Types

### Anti-Patterns

**Detection methods:**
1. Inconsistent examples - Files that don't follow the dominant pattern
2. Comment warnings - Grep for `// TODO`, `// FIXME`, `// WARNING`, `// HACK`
3. Layering violations - Imports that skip layers
4. Naming inconsistencies - Files that break conventions

**Format:** Short, actionable statements starting with "Don't..."

### Conventions

**Detection methods:**
1. Consistent patterns across multiple files
2. Common import structures
3. Shared coding idioms

**Format:** Descriptive rules about what TO do

### New Patterns

**Detection methods:**
1. Groups of files following consistent structure
2. Common naming conventions not yet documented
3. Feature areas not covered by existing patterns

**Required fields:**
- `id` - Pattern identifier (snake_case)
- `keywords` - Trigger words
- `file_convention` - Path template
